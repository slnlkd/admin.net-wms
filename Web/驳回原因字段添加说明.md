# 驳回原因字段 ApprovalRemark 添加说明

## 概述

为物料审核功能添加驳回原因字段 `ApprovalRemark`，使管理员在驳回申请时可以填写驳回原因，业务员可以查看驳回原因。

```typescript
const approvalMaterial = (row: any, approvalStatus: any) => {
  const actionText = approvalStatus === 1 ? '审核通过' : '驳回';
  
  if (approvalStatus === 2) {
    // 如果是驳回，显示输入框让用户输入驳回原因
    ElMessageBox.prompt('请输入驳回原因', '驳回申请', {

      inputPattern: /^.{1,500}$/,
      inputErrorMessage: '驳回原因不能为空且不超过500字符'
    }).then(async ({ value }) => {
      await doApproval(row, approvalStatus, value);
    }).catch(() => {});
  } else {
    // 如果是通过，直接确认
    ElMessageBox.confirm(`确定要${actionText}吗?`, "提示", {
      confirmButtonText: "确定",
      cancelButtonText: "取消",
      type: "warning",
    }).then(async () => {
      await doApproval(row, approvalStatus, '');
    }).catch(() => {});
  }

// 执行审核操作
const doApproval = async (row: any, approvalStatus: any, approvalRemark: string) => {
  await wmsBaseMaterialPresetApi.approvalMaterial(row.id, approvalStatus, approvalRemark);
  
  // 发送消息通知给申请人
  await sendApprovalNotification(row, approvalStatus, approvalRemark);
  
  // 发送SignalR事件，通知审核状态变化
  await sendApprovalSignalREvent(row, approvalStatus);
  
  ElMessage.success("操作成功");
  handleQuery(); // 重新查询
  emit('refresh'); // 通知父组件刷新待审核数量
  
  // 如果审核通过，通知父组件刷新物料信息列表
  if (approvalStatus === 1) {
    emit('refreshMaterialList');
  }
};
```

#### 修改通知函数
```typescript
const sendApprovalNotification = async (row: any, approvalStatus: any, approvalRemark: string) => {
  try {
    const materialName = row.materialName || row.materialCode || '未知物料';
    const applicantName = row.createUserName || '未知申请人';
    const statusText = approvalStatus === 1 ? '审核通过' : '被驳回';
    const currentTime = new Date().toISOString();
    
    let content = `申请人【${applicantName}】申请的物料信息【${materialName}】（编码：${row.materialCode}）已${statusText}。`;
    
    // 如果是驳回且有驳回原因，添加到通知内容
    if (approvalStatus === 2 && approvalRemark) {
      content += `\n驳回原因：${approvalRemark}`;
    }
    
    const noticeData: AddNoticeInput = {
      title: `物料信息审核${statusText}`,
      content: content,
      type: 1, // 通知类型：通知
      status: 1, // 已发布状态
      publicTime: new Date(currentTime), // 发布时间
      publicUserName: '系统管理员', // 发布人
      receiverUserIds: [row.createUserId], // 接收人ID数组
    };
    
    // 添加并自动发布通知
    await getAPI(SysNoticeApi).apiSysNoticeAddPost(noticeData);
    
  } catch (error) {
    console.error('发送通知失败:', error);
    // 通知发送失败不影响主要审核流程
  }
};
```

## 功能说明

### 1. 驳回时输入原因

- 管理员点击"驳回"按钮时，会弹出输入框
- 必须填写驳回原因（1-500字符）
- 点击"确定"后执行驳回操作

### 2. 审核通过

- 管理员点击"通过"按钮时，直接确认
- 不需要输入原因

### 3. 通知内容

- **驳回通知**：包含驳回原因
  ```
  申请人【张三】申请的物料信息【物料A】（编码：MAT001）已被驳回。
  驳回原因：规格不符合要求
  ```

- **通过通知**：不包含驳回原因
  ```
  申请人【张三】申请的物料信息【物料A】（编码：MAT001）已审核通过。
  ```

### 4. 列表显示

- 审核弹窗列表：显示驳回原因
- 申请物料信息列表：显示驳回原因
- 审核通过时驳回原因为空

## 字段说明

| 字段名 | 类型 | 长度 | 说明 |
|--------|------|------|------|
| ApprovalRemark | string | 500 | 驳回原因，可为空 |

## 使用场景

### 场景1：管理员驳回申请

1. 管理员在审核弹窗中点击"驳回"按钮
2. 系统弹出输入框要求填写驳回原因
3. 管理员填写驳回原因："规格不符合要求"
4. 点击"确定"执行驳回操作
5. 系统保存驳回原因到数据库
6. 系统发送通知给业务员（包含驳回原因）

### 场景2：管理员审核通过

1. 管理员在审核弹窗中点击"通过"按钮
2. 系统直接确认通过操作
3. 系统发送通知给业务员
4. 通知内容不包含驳回原因

### 场景3：业务员查看驳回原因

1. 业务员在申请物料信息列表中查看
2. 可以看到驳回状态和驳回原因
3. 驳回原因列会显示具体的驳回内容

## 注意事项

1. **数据库迁移**：需要在数据库中添加 `ApprovalRemark` 列
2. **后端编译**：后端修改后需要重新编译
3. **字段长度**：驳回原因最多500字符
4. **必填校验**：驳回时必须填写原因
5. **通过操作**：通过操作不需要填写原因，传入空字符串即可
