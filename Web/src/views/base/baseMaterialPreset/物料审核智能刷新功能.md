# 物料审核智能刷新功能文档

## 功能概述

物料审核智能刷新功能为业务员和管理员提供了实时、高效的审核状态同步机制，确保双方能够及时获取最新的审核信息，同时通过智能算法减少不必要的页面刷新，优化系统性能。

## 功能特性

### 1. 业务员端 - 智能刷新

#### 核心特性
- **定时查询**：每3秒自动查询一次数据
- **智能对比**：将查询结果与缓存数据进行对比
- **按需刷新**：只有当数据真正发生变化时才刷新页面
- **智能通知**：检测审核状态变化并显示通知

#### 工作流程
```
每3秒查询数据
    ↓
与缓存数据对比
    ↓
数据有变化？
    ↓
    是 → 刷新页面 + 显示通知
    ↓
    否 → 不做任何操作
```

#### 数据对比逻辑
系统会比较以下关键字段：
- `id`：记录唯一标识
- `approvalStatus`：审核状态（0=待审核，1=通过，2=驳回）
- `updateTime`：更新时间

只有当这些字段发生变化时，才会触发页面刷新。

#### 通知机制
当检测到审核状态从"待审核"变为"通过"或"驳回"时，系统会：
- 显示右上角通知
- 通知内容包含物料名称和审核结果
- 通过状态显示绿色通知，驳回状态显示橙色警告

### 2. 管理员端 - 实时通知

#### 核心特性
- **实时监听**：监听新物料申请事件
- **自动刷新**：有新申请时自动刷新待审核数量
- **弹窗同步**：审核弹窗打开时自动刷新列表
- **消息通知**：显示新申请通知提示

#### 工作流程
```
业务员提交新申请
    ↓
管理员页面收到通知
    ↓
自动刷新待审核数量
    ↓
显示右上角通知
    ↓
如果审核弹窗已打开 → 自动刷新弹窗列表
```

### 3. 审核弹窗 - 实时同步

#### 核心特性
- **自动刷新**：审核完成后自动刷新列表
- **实时更新**：有新申请进来时自动刷新
- **无需关闭**：审核完如果有新数据，不用关闭再点开

#### 事件触发
- 审核操作完成后自动刷新
- 收到新申请通知时自动刷新
- 父组件调用 `refreshTable()` 方法时刷新

## 技术实现

### 前端实现

#### 1. 业务员页面智能刷新

```typescript
// 状态管理
const state = reactive({
  tableData: [],
  cachedTableData: [], // 缓存的数据
  lastUpdateTime: null, // 上次更新时间
  refreshTimer: null, // 定时器
});

// 智能刷新函数
const smartRefresh = async () => {
  const newData = await fetchData();
  
  // 检查数据是否发生变化
  const hasChanged = checkDataChanged(newData);
  
  if (hasChanged) {
    // 更新缓存和页面
    state.cachedTableData = JSON.parse(JSON.stringify(newData));
    state.tableData = newData;
    
    // 检查审核状态变化并通知
    checkApprovalStatusChanges(newData);
  }
};

// 数据对比函数
const checkDataChanged = (newData: any[]) => {
  // 比较数量
  if (newData.length !== state.cachedTableData.length) {
    return true;
  }
  
  // 比较关键字段
  for (let i = 0; i < newData.length; i++) {
    const newItem = newData[i];
    const cachedItem = state.cachedTableData[i];
    
    if (newItem.id !== cachedItem.id ||
        newItem.approvalStatus !== cachedItem.approvalStatus ||
        newItem.updateTime !== cachedItem.updateTime) {
      return true;
    }
  }
  
  return false;
};
```

```csharp
[MapHub("/hubs/materialApproval")]
public class MaterialApprovalHub : Hub<IMaterialApprovalHub>
```

5. 管理员可以连续审核多个申请

## 性能优化

### 1. 减少不必要的刷新

- **智能对比**：只有数据变化时才刷新
- **缓存机制**：使用缓存数据减少对比开销
- **按需更新**：只更新变化的记录

### 2. 降低服务器压力

- **减少请求**：3秒查询一次，但不是每次都刷新
- **数据对比**：前端进行数据对比，减少服务器计算
- **批量处理**：多个变化合并处理

### 3. 提升用户体验

- **无闪烁**：避免频繁刷新导致的页面闪烁
- **实时通知**：及时告知用户重要变化
- **流畅操作**：不影响用户正常操作

## 注意事项

### 1. 浏览器兼容性

- 需要支持 WebSocket 的现代浏览器
- 推荐使用 Chrome、Firefox、Edge 等主流浏览器
- IE 11 及以下版本不支持

### 2. 网络要求

- 需要稳定的网络连接
- WebSocket 连接断开会自动重连
- 网络不稳定时可能影响实时性

### 3. 性能考虑

- 大量数据时智能对比可能消耗一定性能
- 建议每页数据量不超过 100 条
- 可根据实际情况调整刷新间隔

### 4. 权限要求

- 需要登录才能使用实时功能
- 需要相应的审核权限
- 租户隔离确保数据安全

## 故障排查

### 问题1：页面不刷新

**可能原因**：
- 网络连接问题
- SignalR 连接断开
- 浏览器不支持 WebSocket

**解决方案**：
1. 检查网络连接
2. 查看浏览器控制台错误信息
3. 检查后端 SignalR 服务是否正常运行

### 问题2：通知不显示

**可能原因**：
- 浏览器通知权限未开启
- 通知被浏览器拦截
- 通知配置错误

**解决方案**：
1. 检查浏览器通知权限设置
2. 确保通知不被浏览器拦截
3. 检查通知代码配置

### 问题3：数据不同步

**可能原因**：
- 缓存数据不一致
- 数据对比逻辑错误
- 定时器未正常工作

**解决方案**：
1. 清除浏览器缓存
2. 检查数据对比逻辑
3. 查看定时器是否正常运行
